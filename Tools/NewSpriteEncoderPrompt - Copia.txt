Analizza ciascuna frame come segue:
- per ciascuna riga:
-  individua il primo pixel non trasparente, se la sua posizione è dispari prendi il pixel pari precedente
-  individua l'ultimo pixel non trasparente successivo a quello individuato, se la sua posizione è pari prendi il pixel dispari successivo
-  i pixel così individuati, unitamente all'ascissa iniziale e finale, codificati in byte (un pixel per nibble), sono denominati "run" 
-  individua tutte le run all'interno della linea
-  se includendo i pixel trasparenti di cui sopra, due run risultano consecutive, uniscile in una sola run

-  individua tutte le run all'interno di tutta la frame corrente

Metti dataFrameKKKK: prima del primo run della frame e termina con una ret il codice della frame

       ld c,0x98
inserisci il codice per tracciare la frame

Metti  prima di ciascun blocco di 4 frame la routine:

.area _SEG100


    .org 0xA000
	
    jp Frame0000
    nop
    jp Frame0001
    nop
    jp Frame0002
    nop
    jp Frame0003
    ret

Dove le label Frame0000-Frame0003 sono le frame del file corrente

All'inizio del blocco successivo usa Frame0004-Frame0007 e così via

_SEG100 è il segmento del mapper rom iniziale 
Nei blocchi successivi usa _SEG101 _SEG102 ecc 

Il nome del file è sprites.asm

Attenzione: se la frame è completamente trasparente (frame vuota) genera soltanto la label ed il ret come segue
FrameKKKK: ret
Fai in modo di non generare altro codice per le frame vuote
